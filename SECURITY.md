# Security Policy

## Supported Versions

Only the current `main` branch is maintained.

## Reporting

For security-sensitive issues, use GitHub private vulnerability reporting if it
is enabled for the repository. If it is not enabled, open a minimal public issue
that describes the affected area without posting private dumps, proprietary
firmware, credentials, or production MAC addresses.

For ordinary crashes, build issues, documentation problems, or hardware recovery
questions, use the issue templates.

## Hardware Safety

`flashwrite` is destructive. A security report should not require a maintainer
or user to run a destructive command before the issue can be understood. Include
read-only reproduction steps whenever possible.
